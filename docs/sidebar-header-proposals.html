Sidebar Header 実装仕様(案E採用)

検討した5案(powerline バッジ / タブ+罫線 / タイトルバンド / 分布メーター / バッジ+チップ)のうち 「案E: powerline バッジ + フィルタチップの2段構成」を採用。 さらに「0件フィルタは適用不可」「適用中に0件になってもヘッダーを維持」の2ルールを組み合わせる。 ターミナル部分は実機のダーク配色・ratatui で表現可能な範囲(セル単位の fg/bg・bold・Unicode グリフ)で描いている。

RULES

インタラクションルール

  1. カウントは常にフィルタ適用前の全体から算出する。フィルタ中でも他状態の件数が正しく見える(現状はフィルタ後 rows から算出しているため壊れている)。
  2. 0件のフィルタは適用できない。0件チップは dim 表示・クリック無効。tab サイクルも0件状態をスキップする。all は常に適用可。
  3. 適用中に0件になっても自動解除しない。アクティブチップは状態色反転のまま ▲ 0 と表示し、リスト領域にフィルタ文脈付きの空状態を出す。

NOTES

実装上の制約と表現

  • powerline 矢印: nerd font の (U+E0B0)。fg=直前セグメント bg、bg=直後セグメント bg(行末は default bg)。
  • チップ: 前後1スペースの矩形 bg 塗り。HTML 上の角丸は演出であり実機は矩形でよい。丸めたい場合のみ /(U+E0B6/U+E0B4)を使う(初期実装ではやらない)。
  • アクティブチップの fg: 端末背景色に近い暗色(bg との反転)。theme に反転用 fg を追加する。
  • 0件チップ: bg なし・marker 色(darkgray)。枠線は端末では表現しないので dim テキストのみ。
  • 幅フォールバック: 幅が足りない場合は 1行目の総数セグメント( 7 tasks )から省略する。チップははみ出し時に既存の truncate_display で切る。
