vde-tmux / top status bar — window list

上部バー ウィンドウ一覧の視認性改善 — 複数案

tmux 上部バーの session/window 一覧について、指摘された 3 つの痛点 ①要素の境界が溶ける ②状態色が似ていて区別できない ③badge を塗りの中に置くか外に置くか を分けて複数案を出します。バー地は実測の #232332。コントラスト比(CR)は WCAG 2.x 相対輝度で実算出。 緑枠が推奨案です。

現状の痛点 ① 境界を立てる ② 色を離す ③ badge の塗り内/外 推奨バー(統合) 実装マッピング
核となる原則 信号グリフ(状態●▲✓○)は常に暗いバー地の上に置く。明るい塗り(面)は名前ラベルにだけ使う。 塗りとグリフを重ねると、明るい面の上でグリフの状態色が潰れる(実測 CR 1.1 前後)。この分担が 3 論点すべての土台になります。

00現状 — 痛点の確認

badge_style = inline / format = " {session} " をセパレータなし("")で連結。 グリフは状態色、名前は既定色、カレントのみ bold。

痛点は 3 つ。(a) 隣の session との間に区切りが無く、名前が長いと miyauchidp.dev と vde-monitor が地続きに見える。 (b) working の緑 #4fd08a と done の水色 #45cbe6 は輝度がほぼ同じ(CR 1.02)で、色だけでは見分けられない。 (c) カレントに塗りを足したくても、塗りの上にグリフを載せると潰れる。

01① 要素の境界を立てる

セグメント間に区切りを与える。幅コストと主張の強さでトレードオフ。

A1

細いディバイダ │(最小変更)

推奨

thin divider

● public│ ✓ miyauchidp.dev│ ● vde-monitor│ ▲ vde-tmux 1:node
PRO
  • 幅コスト実質ゼロ(区切り1文字)。狭いバーでも破綻しない
  • 塗りを一切足さないので原則(グリフは暗地の上)を自然に守る
  • config だけで実現(セグメント連結に separator を挿入)
CON
  • 面での分離ではないので、bold カレント以外の強調は弱め
  • ディバイダ色を暗くしすぎるとバー地に沈む(#4a4860 程度が下限)
A2

  • 暗いチップなのでグリフ CR ≈ 6 を保てる(明るい pill と違い潰れない)

  • 塗りが常時全面に入り、状態色(赤=詰まり)の視線誘導を殺す
  • カレント塗りの上にグリフが乗ると潰れる(③参照)。塗り前提の設計は原則に反する
① の推奨

より強い面分離が欲しい/session 数が少ない環境では A2(暗いチップ) を選択肢に。明るい pill ではなく暗いチップにするのがグリフを潰さない鍵。

02② 状態色を離して区別できるようにする

現状 working(緑 #4fd08a)と done(水色 #45cbe6)は輝度差 CR 1.02。 形(●/✓)は違うが、遠目や小フォントでは色が同じに読める。色相と輝度の両方を離す。

B1

パレットを色相ごと離す(緑→青へ)

推奨

● working│ ✓ done 改良: 緑と青で明確に別
▲ blocked│ ● working│ ✓ done│ ○ idle

提案値: blocked #ff7a7a / working #57d98a(緑) / done #5aa6ff(青) / idle #8b86a3。 done を水色から青に振ることで working の緑と色相が分離し、輝度差もつく。

PRO
  • 色相(緑↔青)と輝度の両方が離れ、小フォント・色覚特性下でも区別できる
  • グリフ形(●✓▲○)との二重符号化で色に依存しない
  • badge.colors.* の値差し替えのみ。コード変更ゼロ
CON
  • done の「青」が blocked 以外の青系 UI と混同されないか要確認(sidebar の repo 青との整合)
B2

補助

shape-only

◐ working│ ✔ done 形を強く差別化
PRO
  • 配色を一切変えないので既存テーマとの整合の心配がない
CON

  • グリフ形はフォント依存で細部が潰れやすい
② の推奨

B1(パレットを離す)。done を水色→青に振るのが決め手で、working の緑と色相・輝度の両方で分離する。値差し替えだけで済む。

B2 の「形も離す」は B1 と併用すると二重符号化になり、色覚特性への保険になる(必須ではない)。

03③ badge を塗りの「中」か「外」か

カレントを塗り(面)で強調したい場合、状態グリフをその塗りの中に入れるか、塗りの外(バー地)に出すか。実測で決着する論点。

C1

塗りの「中」— 明るい pill にグリフを載せる

非推奨

badge inside bright pill

● public│ ▲ vde-tmux ▲ が塗りに溶ける
blocked ▲ on pill = 1.33 working ● on pill = 1.07 done ✓ on pill = 1.09
PRO
  • 1 セグメントに全情報が収まって見た目はまとまる
CON
  • 明るい pill の上でグリフ状態色が CR 1.1 前後に潰れ、状態が読めなくなる
  • いちばん重要な「詰まり=赤」の警告がカレントで消える本末転倒
C2

塗りの「外」— グリフはバー地、塗りは名前だけ

推奨

badge outside pill (= badge_style: outer)

● public│ ▲ vde-tmux ▲ は暗地の上で鮮明
blocked ▲ on bar = 5.57 名前 on pill(暗字) = 7.39 pill vs バー地 = 7.39
PRO
  • グリフは常に暗地の上 → 全状態色が鮮明(CR 5.5〜8)
  • 塗りは名前ラベル専用にでき、暗字(#232332)で CR 7.4 の可読性
  • badge_style = "outer" が既に実装済み。config 切替のみ
CON
  • グリフが pill の左に一段はみ出すので、+1 列ぶん幅が要る
C3

塗りを使わず bold + 暗チップ

代替

no bright fill

● public│ ▲ vde-tmux 暗チップ内でも読める
glyph on 暗チップ ≈ 5〜6名前(明字) ≈ 7.5
PRO
  • 明るい面を使わないのでグリフを塗りの中に入れても潰れない
  • A2(暗チップ境界)とそのまま両立する
CON
  • カレントの主張は明るい pill(C2)より弱い
③ の推奨

C2(塗りの外)。明るい pill を使うなら、グリフは必ずバー地に出す。これは既存の badge_style = "outer" で今すぐ切替可能。

明るい面自体を避けたいなら C3(暗チップ)。この場合はグリフを中に入れても潰れないので A2 とセットで使える。

①A1 + ②B1 + ③C2 を重ねた本命と、面分離を強めた ①A2 + ②B1 + ③C3 の代替を並べます。

本命

ディバイダ + 離したパレット + カレントは pill(グリフ外)

推奨

recommended

05実装マッピング(ほぼ config)

  1. ③ グリフを塗りの外へ

    statusline.sessions.badge_style = "outer"(実装済み)。カレント塗りは sessions.current.colors.bg = "#98b2f6" / fg = "#232332"。

  2. ② パレットを離す

    badge.colors を blocked #ff7a7a / working #57d98a / done #5aa6ff / idle #8b86a3 に差し替え。コード変更なし。

  3. ① 境界(A1 ディバイダ)

    セグメント連結が現状 join("")。ここに区切り(例 #[fg=#4a4860]│#[default])を挟む小改修が要る。render_statusline_sessions_with_stale の join を separator 付きに。あるいは sessions.other.suffix にディバイダを入れる暫定手もあるが、末尾に余分な区切りが残る点に注意。

  4. ① 境界(A2 暗チップ / 代替案)

    sessions.other.colors.bg = "#2e3350" + format = " {session} "。ディバイダ不要。カレントは current.colors.bg = "#3a3550" + bold。

  5. idle の底上げ

    idle が暗地/チップで沈むので #6f6b85 → #8b86a3(② に含む)。

②③ と代替①(A2)は config 差し替えのみで試せます。本命①(A1 ディバイダ)だけ join への separator 挿入という数行の改修が必要。 まず ②(色)③(outer)を config で入れて効果を確認し、境界を A1/A2 のどちらで固めるか実機で選ぶのが安全です。
