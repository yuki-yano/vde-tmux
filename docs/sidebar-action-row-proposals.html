jump / preview ボタンの配色バランス

現状の ↗ jump は branch と同じ明るいシアン(ハードコードの Color::Cyan)、 ⌕ preview は pin と同じラベンダー(147)。detail ゾーンの控えめなグレーの中で 2 色のフルアクセントが並ぶため浮いて見える。周辺トーンと馴染ませる案を比較する。

現状(参考)

   ● claude  sidebar polish       4m
     prompt: LIVE の色を理想形に…
     session: dotfiles / pane: %12
     ↗ jump   ⌕ preview
 ▾ dotfiles  main   ← branch と jump が同色で衝突

比較案

案 A — ラベルは灰、グリフだけ淡アクセント

     session: dotfiles / pane: %12
     ↗ jump   ⌕ preview
 ▾ dotfiles  main
  • ラベル文字を detail 色(246)にして detail ゾーンへ馴染ませ、グリフ 1 文字だけ淡色(jump=淡シアン 73 / preview=淡ラベンダー 103)で「押せる」を示す
  • branch の明るいシアンとは彩度差ではっきり区別でき、色相の文法(シアン系=移動、ラベンダー系=閲覧・操作)は維持される
  • カーソルが乗ると既存の選択ハイライト(bg 237 + BOLD)で自然に点灯するため、選択時の視認性は落ちない
  • 非選択時のボタンの発見性はいまより下がる(detail 行との差はグリフの色だけ)

案 B — 選択時のみフル点灯

     session: dotfiles / pane: %12
     ↗ jump   ⌕ preview   ← 非選択時

     ↗ jump   ⌕ preview        ← 選択時
  • 非選択時は完全に detail 扱いなので浮きが一切ない。fisheye の「選択に情報が集まる」思想と一貫する
  • 選択時は現行のフルアクセントが復活し、キーボード操作の直前に一番目立つ
  • マウス派にはクリック可能であることが選択するまで分からない
  • 選択状態でスタイルが切り替わる実装(render に selected 分岐追加)が必要で、行の見た目が状態依存になる

案 C — 操作色(ラベンダー)に統一

     session: dotfiles / pane: %12
     ↗ jump   ⌕ preview
 ▾ dotfiles  main
  • jump / preview とも淡ラベンダー(103)にし、jump のみ BOLD で主アクションを示す。branch シアンとの衝突が構造的に消える
  • pin ✦・active バー・mode と合わせて「ラベンダー=vde の操作系」という色の文法が確立する
  • jump と preview の区別が太さだけになり、2 ボタンの判別は案 A より弱い
  • ラベンダーの出現箇所が増え、pin や active バーの希少性(=目立ち)が薄まる

案 D — ブラケットで形にして色を抜く

     session: dotfiles / pane: %12
     [↗ jump] [⌕ preview]
 ▾ dotfiles  main
  • marker 色のブラケットが「ボタンである」ことを形で示すため、色をほぼ抜いても発見性が残る(案 A の弱点を補う)
  • クリック当たり判定の範囲がブラケット込みで視覚と一致する
  • 幅を 4 桁余分に使う(40 列では許容範囲だが detail ゾーンがややうるさくなる)
  • サイドバー内に括弧の記法がここにしか無く、新しい視覚語彙が増える

推奨: 案 A。detail ゾーンの中では「グリフ 1 文字の淡色」だけで十分ボタンとして読め、 選択が乗れば既存の選択ハイライトが点灯を肩代わりする(案 B の利点を実装追加なしで半分得られる)。 branch との衝突は色相ではなく彩度で解くため、「シアン系=移動先」の直感も壊さない。 発見性をさらに補いたくなったら案 D のブラケットを後付けできる(A と D は排他ではない)。

実装影響(案 A): `render.rs` の Jump 行描画のみ。4 span 化(`↗`=Indexed(73) / `jump`=theme.detail / `⌕`=Indexed(103) / `preview`=theme.detail)。 列 hit-test(`jump_row_action_at`)はレイアウト不変なので変更なし。config キーは追加せず既定値のみ(要望が出たら `colors.action_*` を検討)。
